ComformHex Tutorial

          

            

              
              
            

            
          

        

        

          

            Recommended workflow: use the GUI to discover the selection and refinement process on a coarse model, then export the command script and replay it offline on a denser grid.
          

          

            

              
Quick Start

              
    
                
  1. Generate a coarse grid with small nx, ny, nz.
    2. 
                
  2. Use B, O, or the coordinate box controls to select cells.
    3. 
                
  3. Preview the refinement first, then apply it.
    4. 
                
  4. Save important regions as cell sets or node sets.
    5. 
                
  5. Export VTK, INP, or the command script when the process looks right.
    6. 
              

            

            

              
View And Selection

              
    
                
  • R: rotate the view. Use the R:X / R:Y / R:Z selector for axis-constrained rotation.
    • 
                
  • P: pan the model in screen space.
    • 
                
  • Z: drag a zoom box. Mouse wheel zooms continuously.
    • 
                
  • B: hold and drag for rectangle selection.
    • 
                
  • O: drag from center outward for circle selection.
    • 
                
  • Click a visible cell to pick it. Use Alt+click to remove a picked cell from the selection.
    • 
                
  • H: hide selected cells and enter Hide Pick mode. With no selection, H toggles Hide Pick mode. Shift+H: restore hidden cells.
    • 
              

            

            

              
Refinement Flow

              
    
                
  1. Keep the first pass coarse so view refresh stays light.
    2. 
                
  2. Prefer box-based or full-set selection when you know the region in model coordinates.
    3. 
                
  3. Use Focus if you want several rounds of finer local picking.
    4. 
                
  4. After the refinement shape is stable, create sets and assign material names.
    5. 
                
  5. Run export only after the process is settled.
    6. 
              

            

            

              
Diagnostics

              
    
                
  • Check Hanging runs the conformance check.
    • 
                
  • BC Face cycles through off, all boundary/unmatched faces, and only hanging-report faces.
    • 
                
  • Use slice, hide, or focus together with BC Face to inspect internal transition or hanging-node faces.
    • 
              

            

            

              
GUI To Offline

              
    
                
  1. In the GUI, export the command script after the workflow is validated.
    2. 
                
  2. Build the project once with npm run build.
    3. 
                
  3. Replay offline with a denser grid:
                  node scripts/run-offline-auto.mjs workflow.json --nx 128 --ny 64 --nz 64 --selection replay --vtk output/refined.vtk --inp output/refined.inp
                
    4. 
                
  4. Use selection replay when the script depends on coordinate-box or other mesh-space selection logic.
    5. 
              

            

          

          

            
Current Limit

            

              Offline density upscaling is most reliable when the workflow uses mesh-space selection such as coordinate boxes, select all, invert, clear, hide, and show hidden. Screen-space rectangle, circle, and single-cell picks are good for exploration, but they are not the strongest basis for automated dense replay yet.

            推荐流程：先用 GUI 在较粗的网格上摸清选择与细化流程，确认结果后，再把命令脚本导出，用更密的网格离线重放。
          

          

            

              
快速开始

              
    
                
  1. 先用较小的 nx、ny、nz 生成粗网格。
    2. 
                
  2. 用 B、O 或坐标框选控件选择单元。
    3. 
                
  3. 先看 Preview，确认后再点 Refine。
    4. 
                
  4. 把重要区域保存成 cell set 或 node set。
    5. 
                
  5. 结果稳定后再导出 VTK、INP 或命令脚本。
    6. 
              

            

            

              
视角与选择

              
    
                
  • R：旋转视角；可用 R:X / R:Y / R:Z 限制绕某个轴旋转。
    • 
                
  • P：按屏幕方向平移模型。
    • 
                
  • Z：拖框缩放；鼠标滚轮可连续缩放。
    • 
                
  • B：按住后拖拽，进行矩形框选。
    • 
                
  • O：从中心向外拖拽，进行圆形选择。
    • 
                
  • 直接点击可 pick 一个可见单元；Alt+点击可从选择中移除该单元。
    • 
                
  • H：隐藏已选单元并进入 Hide Pick 模式；没有选择时，H 用于开关 Hide Pick。Shift+H：恢复隐藏单元。
    • 
              

            

            

              
细化流程

              
    
                
  1. 第一轮尽量保持粗网格，视图刷新会更轻。
    2. 
                
  2. 如果区域坐标明确，优先使用 box 或全局集合类选择。
    3. 
                
  3. 如果要逐步缩小局部区域，可以打开 Focus。
    4. 
                
  4. 细化范围稳定后，再保存 set 并指定材料名。
    5. 
                
  5. 确认流程稳定后再导出。
    6. 
              

            

            

              
诊断显示

              
    
                
  • Check Hanging 执行网格连续性检查。
    • 
                
  • BC Face 可在关闭、全部边界/未匹配面、仅 hanging 报告面之间切换。
    • 
                
  • 配合 slice、hide 或 focus 使用，可以观察内部补连续面或 hanging-node 面。
    • 
              

            

            

              
从 GUI 到 Offline

              
    
                
  1. 在 GUI 中验证流程后，导出命令脚本。
    2. 
                
  2. 先执行一次 npm run build。
    3. 
                
  3. 然后用更密的网格离线重放：
                  node scripts/run-offline-auto.mjs workflow.json --nx 128 --ny 64 --nz 64 --selection replay --vtk output/refined.vtk
                
    4. 
                
  4. 如果脚本依赖坐标框选等 mesh-space 逻辑，优先使用 selection replay。
    5. 
              

            

          

          

            
当前限制

            

              离线放大重放目前最稳定的是 mesh-space 选择，例如 coordinate box、全选、反选、清空、隐藏和恢复隐藏。屏幕矩形、圆形和单元 pick 适合探索，但用于高密度自动重放时仍需要谨慎。
